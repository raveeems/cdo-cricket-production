WHY THESE TWO CHANGES WORK TOGETHER
─────────────────────────────────────────

After these changes the impact auto-detection flow is:

Step 1 — Admin taps ⚡ ON manually
  → officialImpactSubUsed = true in DB
  → resolveImpactSlot returns that player
  → correct points calculated
  → Admin override always wins

Step 2 — Admin unavailable, CFLL is primary source
  → fetchCFLLScorecard populates battedOrBowledPlayers from batting/bowling rows
  → heartbeat checks battedOrBowledPlayers for each impact candidate
  → if candidate appears → officialImpactSubUsed = true written automatically
  → correct points calculated

Step 3 — CFLL fails, Cricbuzz takes over
  → fetchCricbuzzScorecard now populates battedOrBowledPlayers from scard loops
  → same auto-detection fires correctly
  → correct points calculated

Step 4 — Admin overrides after the fact
  → Admin taps ⚡ OFF on wrong player, ⚡ ON on correct player
  → Admin taps Recalc
  → all team totals corrected immediately

Step 5 — Both CFLL and Cricbuzz fail (extreme edge case)
  → battedOrBowledPlayers is empty
  → auto-detection does not fire
  → impact player gets zero impact points
  → Admin must set manually when available
  → Admin taps Recalc to correct

This is the correct priority order:
Admin manual > Auto-detection from scorecard > Zero (admin corrects later)
