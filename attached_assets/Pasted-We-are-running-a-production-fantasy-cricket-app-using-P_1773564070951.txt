1. Implement PostgreSQL Connection Pool

Use the official pg pool instead of direct client connections.

Required pool configuration:

max: 10
idleTimeoutMillis: 30000
connectionTimeoutMillis: 2000

Example structure:

create pool
export pool
use pool across all queries

Ensure the pool is singleton so only one pool exists across the server.

When the server starts:

Attempt PostgreSQL connection

If it fails, retry up to 10 times

Wait 3 seconds between attempts

Log each retry

Only start the API server after a successful DB connection.

Example flow:

start server
try connect DB
retry if failed
start API only after success

If the database becomes temporarily unavailable:

The server must not crash

API endpoints should return safe responses

Example response:

{
  error: "Database temporarily unavailable"
}

Add try/catch blocks around all database queries.

4. Add a Health Check Endpoint

Create endpoint:

/health

Response when healthy:

{
 server: "running",
 database: "connected"
}

Response if DB fails:

{
 server: "running",
 database: "disconnected"
}

Verify the backend always reads the database connection string from:

process.env.DATABASE_URL

Add query timeout protection so slow queries do not freeze the server.

Recommended:

statement_timeout = 5000
